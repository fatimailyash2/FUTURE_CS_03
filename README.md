# FUTURE_CS_03

This task presents an API security risk analysis performed on the ReqRes API, a publicly available RESTful service designed for testing and learning purposes. The analysis was conducted by interacting with multiple API endpoints using Postman, sending GET and POST requests to observe response behavior, headers, authentication handling, and input validation mechanisms.

This report contains a detailed examination of several API endpoints, focusing on key aspects such as request and response headers, authentication mechanisms, rate limiting policies, error handling, and data exposure. Each endpoint is analyzed to identify potential security weaknesses, misconfigurations, and abnormal behaviors that may indicate underlying vulnerabilities.

The report highlights common API security risks, including improper input validation, inconsistent rate limiting, excessive data exposure through headers, weak authentication and token handling, and publicly accessible endpoints without authorization. These findings demonstrate how even simple APIs may contain security gaps if not properly configured.

Additionally, this report provides insights into best practices for securing APIs, including enforcing strict validation, implementing consistent rate limiting, minimizing information disclosure, and strengthening authentication mechanisms. The analysis contributes to a better understanding of API security fundamentals and emphasizes the importance of proactive testing in identifying and mitigating potential threats.
